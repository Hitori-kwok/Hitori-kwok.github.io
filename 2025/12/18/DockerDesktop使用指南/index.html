Docker 使用教程

Docker 的优点

    1. ⼀致性和可移植性:Docker 容器可以在任何⽀持 Docker 的地⽅运⾏,⽆论是开 发者的本地机器、 测试环境还是⽣产环境。
    1. 隔离性:每个容器相互隔离,确保应⽤程序之间不会互相⼲扰。
    1. 效率:容器是轻量级的,它们共享操作系统内核,启动速度快,占⽤资源少。
    1. 版本控制和管理:Docker 允许你为应⽤程序创建和管理不同的版本。

Docker 的基本概念

    1. 镜像(Image):Docker 镜像是⼀个只读的模板,⽤于创建 Docker 容器。镜像可 以包含操作系 统、应⽤程序及其依赖项。
    1. 容器(Container):容器是由镜像创建的运⾏实例。它是⼀个轻量级、独⽴的执 ⾏环境。
    1. Dockerfile:这是⼀个⽂本⽂件,包含⼀系列指令,⽤于⾃动化地构建 Docker 镜 像。
    1. Docker Hub:这是⼀个公共的镜像仓库,可以从这⾥下载官⽅和社区维护的 Docker 镜像。
